步骤简述
1.搭建环境
首先安装好必须的运行库，node.js、git、npm, 然后在github 上建立自己的账号
建立好自己的库（库名Repository name注意为：github用户名.github.io；记得添加readme.md文件）
建立好库之后，可以打开setting，可以看到Github Pages一栏，之后我们可以在Custom domain更改域名

注意：库名一定要是文中所述格式，否则之后的访问会失效！

2.安装Hexo
本地安装新建一个文件夹用来存放 Hexo 的程序文件，如 Hexo-Blog.打开该文件夹，右键->Git Bash Here.在git内使用 npm 一键安装 Hexo 博客程序：
$npm install -g hexo-cli
注意，可能界面会没有反应，因为下载速度非常缓慢！！！所以不需要担心，静静等待即可，如果过长时间无响应，请重新运行指令安装

初始化并安装所需组件：
打开刚才建立的Hexo-Blog，在文件管理器的地址栏（我的地址栏显示的是D:\hexo\kagaminejin\）输入cmd，然后回车，打开cmd命令
hexo init # 初始化
npm install # 安装组件
完成后依次输入下面命令，启动本地服务器进行预览：
hexo g # 生成页面
hexo s -p 5555 # 启动预览
访问http://localhost:5555 ,出现Hexo默认页面,本地博客安装成功！

3.部署Hexo到GitHub Pages
首先安装 hexo-deployer-git：
npm install hexo-deployer-git –save
然后修改 _ config.yml 文件末尾的Deployment部分,修改成如下：
deploy:
type: git
repository: git@github.com:用户名/用户名.github.io.git
branch: master

之后的每一次更新都需要进行以下步骤
更新完成后运行
hexo g
生成页面后
hexo d
将网站上传部署GitHub Pages.
这时访问我们的GitHub(域名 https://用户名.github.io )就可以看到Hexo网站了.

4.完成博客搭建
使用的是 GitHub 的子域名，即用户名.http://github.io ,我们可以为Hexo博客绑定自己的域名替换GitHub域名，更加个性化和专业，也利于SEO.
我们使用Namesilo进行注册，便宜好用没啥套路，使用优惠码okoff优惠1$,.com 域名大概 50 块一年,在国内大概55r一年(阿里云什么的).

5.使用自己的博客
进入博客所在目录，右键打开Git Bash Here,创建博文：
hexo new “My New Post”
然后 source 文件夹中会出现一个My New Post.md文件，就可以使用Markdown编辑器在该文件中撰写文章了.
也可以不使用命令自己创建 .md 文件，只需在文件开头手动加入如下格式Front-matter即可，同上写完后运行hexo g和hexo d发布

title: Hello World # 标题
date: 2019/3/26 hh:mm:ss # 时间
categories: # 分类

  • Diary
    tags: # 标签
  • PS3
  • Games

网站设置包括网站名称、描述、作者、链接样式等，全部在网站目录下的 _ config.yml 文件中，参考官方文档按需要编辑.
（注意：冒号后要加一个空格!）
更换主题在Hexo主题站选择一个喜欢的主题，比如 NexT，进入网站目录打开Github下载主题到本地，将其移动到[E:\hexo\kagaminejin\themes],
然后修改 _ config.yml 中的 theme 为新主题名称 next，发布。（有的主题需要将 _ config.yml 替换为主题自带的，参考主题的文件夹名.）

常用命令
hexo new “name” # 新建文章
hexo new page “name” # 新建页面
hexo g # 生成页面
hexo d # 部署
hexo g -d # 生成页面并部署
hexo s # 本地预览
hexo clean # 清除缓存和已生成的静态文件
hexo help # 帮助

6.注意事项
可能你会发现你的rss没有使用过，可以参考我在制作过程中看到的修正方法：
“因为在2020年10月1日起，GitHub默认主分支从master更名为main，以上的提交方式会默认创建一个master分支
为保持一致性，可将本地git init的时候默认分支修改为main.
执行：
git –version 查看版本
git config –global init.defaultBranch main
git分支在2.28.0上，重新设置默认分支为main,
最后修改deploy那里的branch：将master改为main,
此方法用于解决本地搭好了主题，在GitHub上却没有变化的问题，亲测有效.
