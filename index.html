7.插入图片

1.将主页配置文件_config.yml 里的post_asset_folder:这个选项设置为true

2.在你的hexo目录下执行npm install hexo-asset-image –save，这是下载安装一个可以上传本地图片的插件

3.等待一小段时间后，再运行hexo n “xxxx”来生成md博文时
/source/ _ posts文件夹内除了xxxx.md文件还有一个同名的文件夹(没有同名文件夹也可以自己新建)

4.最后在xxxx.md中想引入图片时，先把图片复制到xxxx这个文件夹中，然后只需要在xxxx.md中按照markdown的格式引入图片：
![你想输入的替代文字](xxxx/图片名.jpg)
注意： xxxx是这个md文件的名字，也是同名文件夹的名字。只需要有文件夹名字即可，不需要有什么绝对路径。你想引入的图片就只需要放入xxxx这个文件夹内就好了，很像引用相对路径

5.最后检查一遍，hexo g生成页面后，进入public\YYYY\MM\DD\index.html文件中查看相关字段，你会注意到，html标签内的语句是<img src="YYYY/MM/DD/xxxx/图片名.jpg">，而不是<img src="xxxx/图片名.jpg>。

以下是步骤简述

1.环境搭建

首先安装好必须的运行库，git、npm&node.js, 然后在github 上建立自己的账号
建立好自己的库（库名Repository name注意为：github用户名.github.io；记得添加readme.md文件）
建立好库之后，可以打开 setting ，可以看到 Github Pages 一栏，之后我们可以在 Custom domain 更改域名

注意：库名一定要是文中所述格式，否则之后的访问会失效！

2.安装Hexo

本地安装新建一个文件夹用来存放 Hexo 的程序文件，如 Hexo-Blog .打开该文件夹，右键->Git Bash Here.在git内使用npm一键安装 Hexo 博客程序：
$ npm install -g hexo-cli
注意，可能界面会没有反应，因为下载速度非常缓慢！！！所以不需要担心，静静等待即可，如果过长时间无响应，请重新运行指令安装

npm install hexo-deployer-git --save

这时访问我们的 GitHub (域名 https://用户名.github.io )就可以看到 Hexo 网站了.

4.绑定域名

使用的是 GitHub 的子域名，即用户名.http://github.io ,我们可以为Hexo博客绑定自己的域名替换 GitHub 域名，更加个性化和专业，也利于 SEO .
我们使用 Namesilo 进行注册，便宜好用没啥套路，使用优惠码okoff,优惠1$, .com 域名大概 50 块一年,在国内大概55r一年(阿里云什么的).

5.开始使用

进入博客所在目录，右键打开 Git Bash Here ,创建博文：
hexo new "My New Post"
然后 source 文件夹中会出现一个 My New Post.md 文件，就可以使用 Markdown 编辑器在该文件中撰写文章了.
